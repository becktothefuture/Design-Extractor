# Capture Workflow

This library is for collecting useful design ideas from real websites, images, and interactions.

The intended prompt is simple:

```text
Look at this scroll module on this website and add it to my design patterns.
```

The capture workflow then:

1. opens the source in a browser
2. records screenshots or GIFs of the important behavior
3. writes a reusable pattern page
4. tags it with one to five searchable tags
5. checks that the media and links work
6. publishes the updated library

Public pattern pages use real captured screenshots, GIFs, or video. They do not use generated diagrams as substitutes for evidence.

The internal extraction notes stay in the repository so future agents can audit or extend a capture, but the public library is intentionally simpler: search, open a pattern, reuse the idea.
