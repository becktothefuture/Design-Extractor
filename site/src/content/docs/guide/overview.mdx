# Documentation System

The repository remains the source of truth. `knowledge/` stores source reports, findings, patterns, and indexes. `media/` stores the captured evidence. The `site/` project generates a static public library from those folders.

## Architecture

```text
knowledge/        # canonical extraction brain
media/            # canonical evidence media
site/             # generated documentation frontend
```

## Build Flow

```text
site/tokens/*.json
  -> Style Dictionary
  -> site/src/styles/tokens.css
  -> Starlight variables + custom browse pages

knowledge/ + media/
  -> site/scripts/sync-knowledge.mjs
  -> generated Starlight docs + browse JSON + public media
```

Patterns are the main public-facing unit. Reports and moments provide proof boundaries; findings provide atomic observations; recipes provide implementation-ready guidance.

## How To Read The Library

Start with **Patterns** when you want reusable guidance. Open the linked extract report when you need the evidence, limits, captured media, or confidence notes behind that guidance.

Use **Moments** for recorded proof and **Recipes** when translating an observed behavior into implementation constraints. Categories are a secondary index for finding related material across separate extractions.
