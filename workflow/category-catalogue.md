# Category Catalogue

Use these fixed categories for internal extraction evidence. Add a new category only when it creates real retrieval value and document it here first.

Categories are not the public browsing model. Public pattern discovery is driven by search and one to five human-readable tags on each pattern.

## Categories

- `loading-entry`: load states, entry gates, intro sequencing, preloaders, first meaningful interaction.
- `sound-interaction-audio`: sound toggles, interaction audio, ambience, audio feedback, audio timing, mute states.
- `scroll-navigation`: scroll physics, smooth scrolling, scroll-triggered state changes, navigation flow.
- `motion-choreography`: animation rhythm, easing, sequencing, transitions, object motion, kinetic hierarchy.
- `visual-style`: palette, texture, contrast, image treatment, surface quality, atmosphere, visual density.
- `typography`: typeface, type scale, weight, spacing, line-height, expressive text behavior.
- `layout-grid-composition`: grid, margins, alignment, focal placement, layering, responsive composition.
- `project-sequencing`: ordering, project-to-project logic, content progression, narrative flow.
- `hover-touch-feedback`: pointer states, hover reveals, active states, touch adaptations, cursor behavior.
- `media-handling`: image/video treatment, lazy loading, posters, playback, cropping, aspect ratio, compression.
- `photography-framing`: camera angle, crop, focal plane, lens feel, foreground/background separation, occlusion, negative space, and subject placement inside photographic frames.
- `subject-staging`: human/object pose, gesture, prop handling, product-in-use staging, body-as-context, and how subjects make a designed object feel situated.
- `background-webgl-canvas-svg`: shader, WebGL, canvas, SVG, procedural, and layered background behavior.
- `content-model`: source data shape, CMS clues, taxonomy, metadata, repeatable content structures.
- `performance-responsiveness`: load behavior, perceived performance, breakpoints, reduced motion, device handling.
- `reusable-principles`: higher-level rules that can be reused across sources.

## Category Rules

- Pick one primary category per node.
- Add secondary categories as tags.
- Do not use vague categories such as `misc`, `general`, or `nice`.
- If a finding spans categories, keep the node focused and link related nodes.

## Public Retrieval Tags

Public tags belong on pattern nodes. They should be short words a designer or builder would actually search for.

Good examples:

- `hover`
- `cards`
- `motion`
- `scroll`
- `editorial`
- `photography`
- `product`
- `portfolio`
- `navigation`

Rules:

- Use one to five tags per public pattern.
- Prefer concrete behavior, surface, or medium terms.
- Do not use repo-only labels such as `reusable-principles`.
- Do not use `pattern` as a tag.
- Do not surface internal categories as site navigation.
